= Local LLM Lab

Welcome to the Local LLM Lab. In this lab you will work with a fully
self-contained language model environment running on OpenShift -- no
external API keys, no data leaving the cluster.

== What is deployed

Your lab environment includes:

Ollama:: A local model runtime serving *{litellm_model}*. Ollama
manages model weights on a persistent volume and exposes a REST API
inside the cluster.

LiteLLM:: An OpenAI-compatible proxy that sits in front of Ollama.
Every request you send to LiteLLM is forwarded to the local Ollama
instance -- never to an external provider.

== What you will learn

. How to verify the health of the Ollama and LiteLLM services
. How to list available models and send chat completions through the
  OpenAI-compatible API
. How to pull additional models into Ollama and register them with
  LiteLLM
. How a local LLM backend can serve as an offline, AUP-safe inference
  endpoint

== Access details

Every command in this lab uses these values. They are pre-filled in
all code blocks, so you can paste and run without editing.

[cols="1,2,3"]
|===
| Field | Value | Description

| LiteLLM URL
| pass:[<code>{litellm_url}</code>]
| The OpenAI-compatible API endpoint. All `curl` commands target this URL.

| API Key
| `{litellm_api_key}`
| Bearer token for authenticating to LiteLLM.

| Model
| `{litellm_model}`
| The default model served by Ollama behind LiteLLM.
|===
