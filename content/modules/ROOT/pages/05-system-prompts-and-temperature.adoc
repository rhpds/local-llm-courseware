= System Prompts and Temperature

This section uses the fast `{litellm_model_fast}` model. The same
model gives very different answers depending on how you prompt it.
Here you will see how system prompts and temperature settings shape
model output.

== No system prompt

Send a question with no system prompt:

[.execute]
[source,bash,subs="attributes"]
----
curl -s {litellm_url}/v1/chat/completions \
  -H "Authorization: Bearer {litellm_api_key}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "{litellm_model_fast}",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "What is a Pod in Kubernetes?"}
    ]
  }' | python3 -m json.tool
----

Without a system prompt the model defaults to a generic, conversational
tone. The response is likely long and detailed.

== With a system prompt

Same question, but with a system prompt that constrains the format:

[.execute]
[source,bash,subs="attributes"]
----
curl -s {litellm_url}/v1/chat/completions \
  -H "Authorization: Bearer {litellm_api_key}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "{litellm_model_fast}",
    "max_tokens": 1024,
    "messages": [
      {"role": "system", "content": "You are a Kubernetes expert. Answer in one sentence, no preamble."},
      {"role": "user", "content": "What is a Pod in Kubernetes?"}
    ]
  }' | python3 -m json.tool
----

The system prompt compressed the response to a single sentence.
Smaller local models benefit more from explicit system prompts than
large cloud models.

== Temperature comparison

Temperature controls randomness. Low values produce deterministic
output, high values produce creative variation:

[.execute]
[source,bash,subs="attributes"]
----
echo "=== Temperature 0.0 (deterministic) ==="
curl -s {litellm_url}/v1/chat/completions \
  -H "Authorization: Bearer {litellm_api_key}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "{litellm_model_fast}",
    "max_tokens": 512,
    "temperature": 0.0,
    "messages": [
      {"role": "system", "content": "Suggest a creative project name."},
      {"role": "user", "content": "Name a Kubernetes monitoring tool in one word."}
    ]
  }' | python3 -c "import sys,json; print(json.load(sys.stdin)['choices'][0]['message']['content'])"

echo ""
echo "=== Temperature 1.2 (creative) ==="
curl -s {litellm_url}/v1/chat/completions \
  -H "Authorization: Bearer {litellm_api_key}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "{litellm_model_fast}",
    "max_tokens": 512,
    "temperature": 1.2,
    "messages": [
      {"role": "system", "content": "Suggest a creative project name."},
      {"role": "user", "content": "Name a Kubernetes monitoring tool in one word."}
    ]
  }' | python3 -c "import sys,json; print(json.load(sys.stdin)['choices'][0]['message']['content'])"
----

Run it twice -- temperature 0.0 gives the same answer each time,
while 1.2 varies. Use low temperature for code and facts, higher
for brainstorming.

== Summary

* *System prompts* steer tone, format, and behavior
* *Temperature 0.0* for deterministic output, higher for creativity
* Smaller models need more explicit prompting than large cloud models
