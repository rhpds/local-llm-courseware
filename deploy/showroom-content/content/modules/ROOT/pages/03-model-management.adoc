= Model Management

In this section you will learn how models are managed in the Ollama +
LiteLLM stack.

== List models in Ollama

Check which models are currently downloaded:

[source,bash,subs="attributes"]
----
curl -s http://ollama.local-llm-lab.svc.cluster.local:11434/api/tags | python3 -m json.tool
----

You should see *{litellm_model}* along with its size and modification
date.

== Understand the LiteLLM configuration

LiteLLM's configuration is stored in a ConfigMap. The key section is
`model_list`, which maps model names to Ollama backends:

[source,yaml]
----
model_list:
  - model_name: "{litellm_model}"
    litellm_params:
      model: "ollama/{litellm_model}"
      api_base: "http://ollama.local-llm-lab.svc.cluster.local:11434"
----

Each entry in `model_list` registers one model with LiteLLM. To add a
new model, you would:

. Pull the model into Ollama
. Append a new entry to the `model_list` in the ConfigMap
. Restart the LiteLLM deployment to pick up the change

== Model storage

Models are stored on a Persistent Volume Claim (PVC) mounted at
`/ollama-data` inside the Ollama pod. The default PVC size is 10Gi,
which is enough for several small models.

To check current storage usage:

[source,bash]
----
oc exec deployment/ollama -n local-llm-lab -- du -sh /ollama-data
----
